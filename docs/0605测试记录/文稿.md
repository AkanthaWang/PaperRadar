# PaperRadar 视频介绍文稿

大家好，今天简单介绍一下我做的这个项目，名字叫 PaperRadar。

PaperRadar 是一个面向科研论文追踪和分析的自动化工具。它主要解决的问题是：当我们关注某个研究方向时，需要从顶会中查找大量论文、筛选相关主题、下载 PDF、阅读论文并整理总结，这个过程通常比较重复，也比较耗时间。PaperRadar 就是把这些步骤串成一条自动化流程，帮助我们更快地发现和整理有价值的论文。

这个程序主要包含几个功能。

第一，它可以抓取会议论文的元数据，比如论文标题、作者、摘要、关键词、论文链接和 PDF 链接。目前项目支持 OpenReview、CVF、ECVA、DBLP 等来源，可以用于 ICLR、ICML、NeurIPS、CVPR、ICCV、ECCV 等常见会议。

第二，它可以根据关键词筛选论文。比如我关注 emotion、preference 或者其他研究主题，就可以让程序从标题、摘要和关键词中自动筛选相关论文，生成一个筛选后的表格。

第三，它可以批量下载筛选后的论文 PDF，并记录每篇论文的下载状态和本地保存路径。这样后续解析和总结时，不需要手动一个个下载。

第四，它可以调用 MinerU 对 PDF 进行解析，把论文内容转换成 Markdown、图表信息和图片资源。相比直接读取 PDF，这样更方便后续做结构化分析。

第五，它可以调用大语言模型，对解析后的论文生成中文总结报告。报告会尽量整理论文的研究问题、方法、实验结果、创新点和局限，方便快速了解一篇论文的主要内容。

项目也提供了一个本地 Web 前端。启动前端后，可以在浏览器里访问本地页面，查看已经生成的论文报告，也可以选择本地 PDF 文件或者 PDF 目录，直接触发解析和总结任务。前端会显示任务进度、当前处理的文件、运行日志以及最终生成的报告。

从整体流程上看，PaperRadar 可以分成五个阶段：fetch、filter、download、parse 和 summarize。也就是先抓取论文信息，再筛选论文，再下载 PDF，然后解析 PDF，最后生成总结报告。用户既可以分阶段运行，也可以通过一个 run 命令一次性执行完整流程。

这个项目的输出主要包括几类文件：论文元数据表格、筛选后的论文表格、下载好的 PDF、MinerU 解析结果，以及最终的中文 Markdown 报告。这些结果会统一保存在 data 目录下，方便继续查看、整理和复用。

简单来说，PaperRadar 的目标不是替代人工阅读论文，而是把前期最重复的收集、筛选、下载和初步总结工作自动化。这样研究者可以把更多时间放在真正重要的地方，比如判断论文质量、比较方法差异，以及形成自己的研究思路。
